IDE Settings

The easiest way to ensure code conventions is to use your IDE's built-in code formatting utilities.

Most modern IDEs support this functionality, and below you can find Semantico's settings for the most popular IDEs.

IntelliJ

Semantico IntelliJ settings can be downloaded here.

The code style schemes are stored in XML files, in the config/codestyles folder under the IntelliJ IDEA home directory.

On a Mac the path is something like: /Users/yourusername/Library/Preferences/IntelliJIdea11/codestyles/

Once a settings schema is copied to this directory, you can load it in IntelliJ by:

  • Open the Preferences dialog box, and click Code Style
  • In the Code Style page, select Semantico (if you can't see it, even after copying over the file, restart IntelliJ)
  • Apply changes by clicking Apply and then OK on the Code Style page.

To auto reformat a class (or a directory of classes) use control, alt, L (or cmd, alt, L in OSX).

Eclipse/STS

Inconveniently, Eclipse stores formatting and code templating styles in several different files, but we're only concerned with two of them (for now).

The code style schemes are stored in XML files, and can be easily imported within the IDE.

Eclipse Formatter settings can be downloaded here.

Eclipse Clean Up settings can be downloaded here.

To load Formatter settings:

  • Open the Preferences dialog box, and click Java then Code Style, then Formatter.
  • Click on the Import button and select the settings XML (the formatter XML file) you wish to import.
  • Click apply changes by clicking Apply and then OK.

To load Clean Up settings:

  • Open the Preferences dialog box, and click Java then Code Style, then Clean Up.
  • Click on the Import button and select the settings XML (the clean up XML file) you wish to import.
  • Click apply changes by clicking Apply and then OK.

To auto reformat a class (or a directory of classes) use control, shift, F (or cmd, alt, F in OSX).

To clean up a file, go to the Source file menu, and click on the option 'Clean Up'.
