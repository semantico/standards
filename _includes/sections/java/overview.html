Overview

This document contains conventions, rules and guidelines on how Semantico developed products should be coded, formatted and styled.

This document's primary motivation is two- fold: 1) code consistency and 2) best practices. By maintaining consistency in coding styles and conventions, we can ease the burden of legacy code maintenance, and mitigate risk of breakage in the future. By adhering to best practices, we ensure optimized, readable, maintainable code.
